# 产品研发 Skills 索引

本目录包含产品研发流程和角色定义的标准 skill 文档。

## 📋 文档列表

### 流程 Skill

- **[产品研发流程](product-development-flow.md)** - 将一句话需求转化为完整产品的标准流程

### 角色 Skills

#### 需求与设计层

- **[客户代表](roles/customer-representative.md)** - 站在客户视角提出需求
- **[需求分析师](roles/requirements-analyst.md)** - 梳理和扩展需求，系统化分析
- **[产品专家](roles/product-expert.md)** - 产品化分析，形成功能点和形态
- **[UI专家](roles/ui-expert.md)** - 设计符合用户体验的交互界面

#### 开发实现层

- **[前端工程师](roles/frontend-engineer.md)** - 实现前端页面，构建用户交互
- **[后端工程师](roles/backend-engineer.md)** - 实现后端服务，构建业务领域

#### 架构与质量保障层

- **[技术架构师](roles/technical-architect.md)** - 设计系统架构，确保架构质量
- **[测试人员](roles/tester.md)** - 测试系统功能，确保代码正确性

#### 运维与支持层

- **[产品文档专家](roles/product-documentation-expert.md)** - 编写产品文档，让用户易于理解

## 🔄 研发流程概览

```
需求提出 → 需求分析 → 产品化设计 → 界面设计 → 业务实现 → 架构保障 → 测试验证 → 文档交付
   ↓           ↓            ↓           ↓           ↓           ↓           ↓           ↓
客户代表   需求分析师   产品专家    UI专家    前端/后端  技术架构师   测试人员   文档专家
```

## 🎯 使用场景

### 场景1: 从零开始开发一个产品

1. **需求提出**: 客户代表描述需求
2. **需求分析**: 需求分析师扩展和系统化需求
3. **产品化设计**: 产品专家设计产品功能
4. **界面设计**: UI专家设计交互界面
5. **业务实现**: 前端/后端工程师实现功能
6. **架构保障**: 技术架构师确保架构质量
7. **测试验证**: 测试人员验证功能正确性
8. **文档交付**: 产品文档专家编写文档

### 场景2: 开发一个功能模块

1. **需求提出**: 客户代表提出功能需求
2. **需求分析**: 需求分析师分析需求
3. **产品化设计**: 产品专家设计功能点
4. **界面设计**: UI专家设计界面
5. **业务实现**: 前端/后端工程师实现
6. **架构保障**: 技术架构师审查代码架构
7. **测试验证**: 测试人员测试功能
8. **文档交付**: 产品文档专家编写功能文档

### 场景3: 查询角色职责和技能

- 查看各个 role skill 文档，了解角色的核心职责、关键技能、输入输出物和质量标准

## 📚 角色协作关系

### 需求阶段
```
客户代表 → 需求分析师 → 产品专家
   ↓           ↓           ↓
  原始需求    需求分析    产品设计
```

### 设计实现阶段
```
产品专家 → UI专家 → 前端/后端工程师
   ↓          ↓              ↓
  功能清单   UI设计       业务实现
              ↓              ↓
              技术架构师 ←──┘
```

### 测试交付阶段
```
前端/后端工程师 → 测试人员 → 产品文档专家
      ↓              ↓              ↓
   业务实现       测试验证      文档交付
      ↓              ↓
      技术架构师 ←──┘
```

## 🎓 学习路径

### 初学者
1. 先阅读 [产品研发流程](product-development-flow.md)，了解整体流程
2. 然后阅读 [客户代表](roles/customer-representative.md)，了解需求如何提出
3. 依次阅读各个 role skill，了解每个角色的职责和技能

### 进阶者
1. 深入学习 [技术架构师](roles/technical-architect.md) 的架构方法论
2. 学习 [需求分析师](roles/requirements-analyst.md) 的需求分析方法
3. 学习 [测试人员](roles/tester.md) 的测试方法和工具

### 专家
1. 跨角色学习，了解整个研发生态
2. 优化流程，提高研发效率
3. 持续改进，提升产品质量

## 🛠️ 工具和方法

### 需求阶段
- 需求收集：用户访谈、问卷调查
- 需求分析：5W1H分析法、用例分析
- 需求建模：UML、业务流程图

### 设计阶段
- 原型设计：Figma、Sketch、Axure
- 交互设计：交互流程图
- 视觉设计：Photoshop、Illustrator

### 开发阶段
- 前端开发：React、Vue、Angular
- 后端开发：Java、Python、Go、Node.js
- 架构设计：六边形架构、Clean Architecture、DDD

### 测试阶段
- 单元测试：Jest、JUnit
- 集成测试：Selenium、Cypress
- 性能测试：JMeter、LoadRunner
